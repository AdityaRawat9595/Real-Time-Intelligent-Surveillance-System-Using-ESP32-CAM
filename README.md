Real-Time Intelligent Surveillance SystemThis repository contains the implementation of a hybrid, two-tier face recognition pipeline designed for edge-based surveillance. By leveraging the low-cost ESP32-CAM for data acquisition and a host-side hybrid algorithm (Fisherface + FaceNet), the system achieves high accuracy (96%+) without requiring expensive GPUs or cloud-based processing.  ## Project OverviewTraditional surveillance systems often lack real-time intelligence and require manual footage inspection. This project migrates intelligence to the edge, providing a scalable, low-maintenance solution for smart homes, offices, and campuses.  ### Key FeaturesTwo-Tier Recognition: Uses Fisherface for lightning-fast initial passes and FaceNet for deep-level verification.  Hardware Efficiency: Optimized for the ESP32-CAM (under $10 USD), acting as a Wi-Fi-enabled data acquisition module.  Real-Time Feedback: Overlays bounding boxes, recognized names, and confidence scores directly onto the live stream.  Privacy-First: Processed locally on a host machine; no raw images or biometric templates are sent to the cloud.  ## System ArchitectureThe system follows a strategic multi-stage pipeline to balance speed and accuracy:  Face Detection (Haar Cascade): Fast localization of faces in the MJPEG stream.  Tier 1 - Fisherface (Fast Checker): A lightweight classical algorithm (LDA-based) that handles confident matches in under 1ms.  Tier 2 - FaceNet (Deep Thinker): A 128-dimensional embedding model used only when Tier 1 confidence is low, providing resilience against lighting and pose variations.  ## Performance MetricsThe hybrid approach ensures the system stays responsive while maintaining high reliability:  Combined Accuracy: 95% – 97%.  Precision: 93% – 95%.  Latency: Generally below 150ms.  ## Installation & Setup### 1. Hardware RequirementsESP32-CAM Module.  USB-to-Serial Adapter (for flashing firmware).  Host System: Laptop/PC (Intel i5/M2 or better with 8GB RAM).  ### 2. Software PrerequisitesArduino IDE: To flash the ESP32-CAM firmware.  Python 3.x: Recommended via Anaconda environment.  Key Libraries: opencv-python, keras-facenet, tensorflow, scikit-learn, numpy, pandas.  ### 3. Setup StepsFlash ESP32-CAM: Upload the provided Arduino sketch to configure the module as a Wi-Fi IP camera.  Capture Dataset: Run the collection script to gather ~500 images per authorized individual, including an "Unknown" class to reduce false positives.  Train Model: Initialize the Fisherface model and generate FaceNet embeddings for your dataset.  Run Recognition: Execute the main hybrid recognition script to start the live surveillance stream.  ## Decision LogicThe system applies the following rules to every detected face:  Accept immediately if FaceNet cosine similarity is $\ge 0.50$.  Fall back to Fisherface if FaceNet is uncertain; accept if distance is $\le 5500$.  Label as "Unknown" if overall confidence drops below 65%.  ## Future ScopeEdge-Native Inference: Porting quantized models (e.g., MobileFaceNet) directly to the ESP32-S3.  Liveness Detection: Adding anti-spoofing checks to distinguish real humans from photos/deepfakes.  Multi-Factor Biometrics: Integrating voice recognition for enhanced security.
